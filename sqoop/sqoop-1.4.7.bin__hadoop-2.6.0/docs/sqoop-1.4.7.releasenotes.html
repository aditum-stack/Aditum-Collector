Release Notes for Sqoop 1.4.7: December, 2017

This document lists all Sqoop issues included in version 1.4.7 not present in the previous release, 1.4.6.

Bug fixes:

  • SQOOP-3248: Add missing Apache 2 license information to TestCompilationManager.java PostgresqlExternalTableImportTest.java
  • SQOOP-3149: Sqoop incremental import - NULL column updates are not pulled into HBase table
  • SQOOP-3159: Sqoop (export + --table) with Oracle table_name having '$' fails with error (ORA-00942 or java.lang.NoClassDefFoundError)
  • SQOOP-3157: Improve regex introduced in [SQOOP-3152] thus not causing column mapping and AVRO issues
  • SQOOP-3123: Introduce escaping logic for column mapping parameters (same what Sqoop already uses for the DB column names), thus special column names (e.g. containing '#' character) and mappings realted to those columns can be in the same format (thus not confusing the end users), and also eliminates the related AVRO format clashing issues.
  • SQOOP-3152: Fix SqoopOptions#parseColumnMapping, thus Sqoop will support mappings with precisison (e.g. DECIMAL(10,5)) in the future properly
  • SQOOP-3105: Add cleanup logic for PostgreSQL related test cases
  • SQOOP-3140: Removing deprecated mapred.map.max.attempts, mapred.reduce.max.attempts entries and using the new constants directly from Hadoop instead
  • SQOOP-3138: Netezza Direct Import does not support --columns options (Suraj Nayak via Venkat Ranganathan)
